You are building the open-source project `thinktank` — an ensemble AI coding tool. Each iteration of this loop should close ONE issue via a proper PR workflow.

IMPORTANT: Use thinktank itself to generate code. Run `npx tsx src/cli.ts run "<task prompt>" -n 5 -t "npm test"` and then `npx tsx src/cli.ts apply` to apply the best result. If agents time out, that's fine — pick from the ones that succeeded.

## Process for EACH iteration

### Step 1: Assess current state
- Run `gh issue list --state open --limit 30` to see outstanding issues
- Run `git log --oneline -5` to see recent work
- Run `git checkout main && git pull` to ensure you're on latest main
- Pick the highest-priority open issue. Priority order:
  1. Bugs (#59, #60, #66, #68)
  2. Robustness (#54, #63, #69)
  3. High-value UX (#57, #62, #77, #81)
  4. Features and everything else

### Step 2: Create a branch and use thinktank
- `git checkout -b issue-N-short-description`
- Run thinktank with 5 agents and NO timeout override (use default 300s):
  `npx tsx src/cli.ts run "<detailed task prompt referencing files and conventions>" -n 5 -t "npm test"`
- Review the results: `npx tsx src/cli.ts list`
- Preview the recommended agent: `npx tsx src/cli.ts apply --preview`
- If it looks good, apply: `npx tsx src/cli.ts apply`
- If no agent succeeded or the code needs adjustment, implement manually

### Step 3: Verify and fix up
- Run `npx tsc --noEmit` to verify types
- Run `npm test` to verify all tests pass
- Run `npm run lint` to verify lint passes (auto-fix with `npx biome check --write src/`)
- If thinktank's output needs minor adjustments, make them manually

### Step 4: Commit and push
- Stage only relevant files (not .claude/ or .thinktank/)
- Commit with message referencing the issue
- Push with `-u origin branch-name`

### Step 5: Create PR with proper template
Use `gh pr create` with: Summary, Change type, Related issue (Closes #N), How to test, Breaking changes.

### Step 6: Self-review
- Add a review comment noting what you verified
- Note whether the code was generated by thinktank (dogfooding!)

### Step 7: Merge and clean up
- `gh pr merge <number> --squash --delete-branch`
- `git checkout main && git pull`

### Step 8: Comment on the closed issue
- `gh issue comment N --body "Fixed in #PR. Summary of what changed."`

### Step 9: File new issues if needed

## Quality standards
- TypeScript strict mode, no `any`
- All tests must pass before PR
- Lint must pass before PR
- Error messages must be actionable
- Keep functions small and focused
- Add tests for new code paths
